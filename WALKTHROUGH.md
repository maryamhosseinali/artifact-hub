# 5-Minute Walkthrough

A step-by-step walkthrough in lieu of a screen recording. Replace
`<WEB_URL>` / `<MCP_URL>` with the deployed URLs from the top of this repo's
`WRITEUP.md` once live.

## 1. Publish an artifact (web UI) — ~1 min

1. Go to `<WEB_URL>/publish`.
2. Upload any HTML file, image, or PDF. Leave title/description/tags blank.
3. Submit. You're redirected to the artifact detail page — title, description,
   and tags were generated by Claude from the actual file content, and the
   artifact was automatically sorted into a folder (Engineering, Design,
   Product, Marketing, Documentation, or Business) — Smart Auto-Categorization,
   no manual filing step anywhere in the flow.

## 2. Browse and filter — ~30s

1. Go to `<WEB_URL>/`. The artifact you just published appears in the gallery,
   with its auto-assigned folder shown on the card.
2. Filter by type, tag, or folder, or type a natural-language query into
   search (e.g. "something about dashboards") — search is LLM-ranked, not a
   keyword match.

## 3. Comment and summarize feedback — ~1 min

1. Open the artifact, post two or three comments as different "reviewers"
   (any name works, no login).
2. Click "Summarize feedback." Claude reads the comment thread and returns
   themes / action items / disagreements.

## 4. Share link — ~30s

1. On the same artifact page, pick an expiry and click "Create share link."
2. Open the returned `/share/<token>` URL in a private window — it renders
   the artifact with no login. Try a garbage token — you get a 404. (Expired
   links render a distinct "this link has expired" message instead of a 404.)

## 5. MCP server in Claude Desktop — ~1.5 min

1. Claude Desktop → Settings → Connectors → Add custom connector.
2. URL: `<MCP_URL>/mcp` — that's it, no header to configure. Claude Desktop
   discovers the server's OAuth metadata automatically and opens a browser
   tab for a one-time consent screen.
3. On the consent screen, enter the shared access secret (provided
   separately) and click Authorize. Claude Desktop now holds a real
   access/refresh token pair for this server.
4. Start a new chat and ask Claude to, e.g., "list the artifacts in Artifact
   Hub" or "search Artifact Hub for X" — Claude calls `list_artifacts` /
   `search_artifacts` live against the same database as the web app.
5. Ask Claude to "publish an HTML snippet that says Hello World to Artifact
   Hub" — this calls `publish_artifact`, which uploads the content, generates
   metadata (including the auto-assigned folder), and returns a link back
   into the web app you were just looking at.
6. Ask Claude to "list artifacts in the Engineering folder" — this calls
   `list_artifacts` with the new `folder` filter, same taxonomy as the web UI.
7. Ask Claude to "summarize the feedback on artifact <id>" or "create a share
   link for artifact <id> that expires in an hour" to exercise the remaining
   two tools.

That's all 5 MCP tools (`publish_artifact`, `list_artifacts`,
`search_artifacts`, `share_artifact`, `get_feedback_summary`) and all three
LLM features (auto-metadata + Smart Auto-Categorization, feedback
summarization), exercised from both the web UI and Claude Desktop against the
same underlying data.
